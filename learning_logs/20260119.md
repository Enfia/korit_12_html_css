### datalist 태그
- label의 for 속성과 input의 id 속성을 일치시킴
  datalist의 id 속성과 input의 list 속성을 일치시킴

### progress 태그
- 움직이는 로딩바 구현 가능

### meter 태그
- 고정된 로딩바 구현 가능

### iframe 태그 문제점 
1. 보안 문제 : 아이디와 비밀번호를 해킹 당할 수 있음 이를 clickjacking이라고 함
2. 사용성 저하 : 뒤로 가기 등이 적용되지 않음 
3. SEO에 최적화되지 않음 : SEO와 관계가 없음 
4. 브라우저 새창 문제 : 여러 사이트를 불러오는 것과 같아서 RAM을 많이 잡아먹음

### `<div>` vs `<span>` 아무것도 하지 않는 태그들
- CSS 할 때 중요함
- 기능적으로 아무것도 하지 않음

### `<div>`와 `<span>`의 차이점
1. span 태그는 인라인
2. div 태그는 블록

### CSS 적용방법
1. 인라인 방식
- HTML 문서에다가 각각 CSS 적용(tailwind)
2. 내부 스타일 시트
- style 태그에다 작성
3. 링킹 스타일 시트
- HTML과 CSS 파일을 따로 분리
