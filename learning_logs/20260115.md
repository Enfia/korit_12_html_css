## 웹 디자이너 vs 웹 퍼블리셔
웹디자이너가 작업한 결과물을 HTML + CSS 문서로 작성하는 사람을 우리나라에서는 웹 퍼블리셔라는 표현을 씀 
여기에 JS까지 적용하여 기능을 만들어 낼 수 있는 사람을 웹 개발자라고 할 수 있음

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>document</title>
</head>
<body>

</body>
</html>
```
`! + tab` 으로 자동 생성됨
HTML 문서 작성시에 꼭 필요한 내용을 담고 있음
문서 제목, 작성자, 사용 언어 및 각종 메타 정보(데이터) 등으로 구성되어 있음
어차피 형식 자체는 동일하기 때문에 일일이 타이핑할 필요 없이 단축키를 지원함 이를 에밋이라고 하는데
`! + tab`도 이 중 하나에 속함 앞으로 에밋이라고 했을 때, 특정 문서 작성을 빠르게 해주는 단축키로 생각하면 됨
`! + tab`은 .html 파일 생성할 때마다 맨 처음에 하는 작업임
- `<html lang="en">`의 의미
lang 속성은 HTML 문서의 웹사이트가 어떤 언어로 되어있는지 컴퓨터에게 알려주는 역할을 함 이걸 지정해줬을 때 구글/ 네이버 등에서 검색할 때 한글로 된 웹 사이트를 찾는 것이 가능함 또한 시각 장애인들을 위한 컴퓨터가 웹 사이트를 분석할 때도 쓰임 그리고 맥 사용자의 경우 lang="en"인데 캡스락 눌러서 한글로 웹 페이지 작성하면 기본 서체가 궁서체임
- SEO : Search Engine Optimization

## 첫 태그 사용하기
`<어쩌고> + </어쩌고>` : `/`가 있는 쪽은 닫는 태그에 해당함

```html
<div>
  <p>
    안녕하세요.
  </p>
</div>
```

라고 가정했을 때 div 태그 내에 p 태그가 종속되어있다고 볼수 있음
이를 확인할 수 있도록 들여쓰기를 사용하여 다른 개발자들도 쉽게 알아채도록 함

닫는 태그가 없는 태그도 존재함 `<br>`
MDN 공식 문서(HTML 관련) 상에서는 `<br>`을 `<br/>`로 쓰지말 것을 권유하고 있음

근데 리엑트에서는 `<br>`로 쓰면 오류나서 `<br/>`로 써야 함

1. `<hr>` : horizontal의 축약어로 가로줄을 의미함

- `#nbsp;` 강제 공백 / html에서는 스페이스바/ 엔터키 탭키는 코드의 가족성을 위해서만
사용될 뿐이지 웹페이지 자체에는 기능이 없기 때문에 강제로 공백을 삽입해야 할 때 사용 됨

## 제목 태그
`<h1>~<h6>의 태그가 존재함`

그런데 노션 등 일부 마크다운 문법을 사용하는 경우에는

### 문단 태그
`<p>` 태그 : paragraph의 축약어로 html 상에서 문단을 표시하는 역할

### 줄바꿈 태그
`<br>` : 닫는 태그가 필요없음 근데 문단 내에서 굳이 개행해야할 때 사용하기는 하나
코드적으로 얘가 1번 문단과 2번 문단으로 나누기를 원하는 건지 엔터키는 쳤지만 문단이 나뉘어지지 않기를 원하는지 알 수 없기 때문에 사실 권장되지 않음

## 텍스트 관련 태그
웹 페이지에서 텍스트를 지정할 때는 내용에 따라 글자를 강조하거나 중요하다고 표시하게 됨
나무위키에 보면 괜한 뻘글 있는 쪽에 취소선이 그려져 있음 이렇게 특정 텍스트의 스타일을 다양하게 조절하는 태그들도 존재함

### 스타일 강조 태그
제목이 아니라 본문 내에서의 세부 스타일 관련
1. `<b>` 태그 : bold를 뜻함
2. `<strong>` : 굵게를 뜻함
3. `<i>` : 이탤릭체를 뜻함
4. `<em>` : 기울임체를 뜻함
